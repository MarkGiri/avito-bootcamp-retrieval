# Авито. Поиск статей справки по вопросу пользователя

Решение задачи поиска статей справки по пользовательскому вопросу. Для каждого запроса формируется ранжированный список из 10 наиболее подходящих `article_id`.

Итоговый результат на платформе: **MAP@10 = 0.610612**.

Для проверки следует использовать последнюю отправку: она соответствует текущей версии кода и ноутбука в репозитории. Метрика не изменилась - MAP@10 = 0.610612.


## Запуск

Версии библиотек зафиксированы в `requirements.txt`.

Исходные файлы нужно разместить следующим образом:

```text
candidate_data/
├── articles.f
├── calibration.f
└── test.f
```

После выполнения всех ячеек будет создан файл `answer.csv`.

## Подход

HTML обрабатывается с помощью BeautifulSoup. Из него извлекается основной текст, а также отдельно сохраняются заголовки, подписи вкладок и спойлеров, элементы таблиц и атрибуты `alt` изображений.

Для представления текста используются лемматизация, TF-IDF по словам и символьным n-граммам, BM25 и поиск по фрагментам длинных статей.

Итоговая оценка релевантности объединяет два направления:

* модели, обученные на размеченных запросах из `calibration.f`;
* прямой поиск по полному набору статей.

Параметры проверялись с помощью повторной пятикратной кросс-валидации и внефолдовых предсказаний. В качестве метрики использовалась MAP@10. Дополнительно анализировались ошибки для статей, отсутствующих среди меток обучающей части выборки.

## Анализ ошибок

При проверке на `calibration.f` частые статьи нередко занимали почти весь список из 10 результатов. Также при очистке текста терялись отрицания, а длинные статьи плохо сопоставлялись с короткими пользовательскими вопросами.

Кроме того, статьи, отсутствовавшие среди меток обучающей части выборки, получали слишком низкую оценку релевантности.

Для исправления этих проблем были:

* добавлен штраф для слишком частых статей;
* сохранены важные смысловые слова, включая отрицания;
* длинные статьи разделены на пересекающиеся фрагменты;
* добавлен отдельный поиск по всему набору статей.
